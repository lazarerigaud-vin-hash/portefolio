Application de gestion
de projets informatiques

Groupama Centre-Atlantique · Niort 26 mai – 27 juin 2025 C# · ASP.NET Blazor · SQL Server

Contexte et objectif

Durant mon stage de 5 semaines au sein de la DSI de Groupama Centre-Atlantique, j'ai été chargé de développer une application web interne. L'objectif était de centraliser la gestion du cycle de vie des projets informatiques, en remplaçant plusieurs outils disparates (ServiceNow, Lists, Outlook, Teams, Microsoft Project) par une seule interface unifiée.

L'application devait gérer les demandes (création, suivi, priorisation), les projets (intervenants, météo, tâches), les tests et le suivi des temps.

Semaine 1 : Autoformation

La première semaine a été consacrée à l'autoformation via des playlists YouTube de mon tuteur Christophe ASJEME. J'ai découvert :

— C# et .NET : types de données, conditions, boucles, POO, collections, LINQ, async/await

— Sécurité ASP.NET : JWT (JSON Web Tokens), ASP.NET Identity, gestion des rôles et des droits d'accès

Ce format purement vidéo, bien qu'instructif, a montré ses limites sans exercices pratiques. La vraie montée en compétences est venue avec le projet concret.

Semaine 2–5 : Développement

Face à la complexité du cahier des charges, j'ai ciblé un premier cas d'utilisation : l'affichage d'une liste de demandes depuis une base de données.

Étape 1 — Modélisation de la base de données

J'ai créé une table Demandes dans SQL Server via SSMS, avec les champs nécessaires :

-- Création de la table Demande
CREATE TABLE Demande (
    Id           INT PRIMARY KEY IDENTITY,
    NomDemande   NVARCHAR(255),
    Description  NVARCHAR(MAX),
    DateDemande  DATETIME,
    Etat         NVARCHAR(100),
    Direction    NVARCHAR(100)
);

Étape 2 — Modèle C# et connexion EF Core

J'ai créé un modèle Demande.cs et configuré Entity Framework Core pour connecter l'application à SQL Server :

// Program.cs — Configuration de la connexion
using Microsoft.EntityFrameworkCore;

builder.Services.AddDbContext<MonContexte>(options =>
    options.UseSqlServer(
        builder.Configuration.GetConnectionString("MaConnexion")
    ));

Étape 3 — Vue Blazor / Razor

J'ai créé un composant .razor pour afficher les demandes dynamiquement depuis la base de données, en parcourant les données avec une boucle @foreach :

@* Composant Blazor — Affichage des demandes *@
@foreach (var demande in demandes)
{
    <div class="demande-card">
        <h3>@demande.NomDemande</h3>
        <p>@demande.Description</p>
        <span>@demande.Etat</span>
    </div>
}

Difficultés rencontrées

Les principales difficultés ont été la configuration de l'environnement (erreurs de connexion SQL depuis Visual Studio), la compréhension d'un document d'expression de besoin professionnel, et la prise en main d'ASP.NET Blazor sans expérience préalable. Ces obstacles ont été surmontés en décomposant le problème, en utilisant SSMS directement, et en allant chercher les ressources nécessaires.

Bilan et apprentissages

Ce stage m'a permis de réaliser mon premier développement web professionnel complet, de la base de données à l'affichage. J'ai acquis une vision concrète du cycle de développement d'une application en entreprise et confirmé mon intérêt pour le développement logiciel.
